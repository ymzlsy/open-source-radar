# 项目方案 · 信息架构与技术栈

> 技术栈：React + Vite，数据层纯 JSON，Cloudflare Pages 部署，域名 open-source.karaithy.com

## 技术选型

| 层 | 选择 | 理由 |
|----|------|------|
| 前端 | React + Vite + TypeScript | 卡片墙/详情/筛选/趋势图这类数据应用最顺手 |
| 样式 | Tailwind CSS | 与现有工作流一致，快速搭卡片/bento |
| 数据 | 静态 JSON（`/data` 直接打包或 fetch） | 零后端，cron 写文件 + git 即更新 |
| 趋势图 | 轻量 sparkline（如 自绘 SVG / recharts 按需） | star 趋势小图 |
| 路由 | React Router | 列表→详情 |
| 部署 | Cloudflare Pages，git push 自动 | 复用 karaithy 既有模式 |

> 数据与代码同仓：`/data` 是 SSOT，cron 改 `/data` 后 push 即触发部署。

## 站点地图

```
/                     首页：4 模块概览
/developers           我的开发者：全部卡片 + 标签/tier 筛选
/developers/:id       开发者详情：简介/事迹/动态时间线/项目/商业化/我的判断
/recommended          推荐开发者：阵地卡片 + 一键晋升入口
/projects             热门开源项目流：列表 + 筛选 + star 趋势
/projects/:id         项目详情：基本信息/star 趋势/深度分析/原项目地址/复刻意向
/articles             热门文章流：列表 + 类型筛选
/articles/:id         文章详情
```

## 首页布局（模块对应）

1. **Hero**：一句话说明这个站是什么 + 今日更新计数（X 个新项目 / Y 篇新文章）。
2. **模块1 · 我关注的开发者**：core tier 优先的卡片墙。卡片正面 = 头像 + 名字 + oneliner + 阵地图标 + 代表作 star。点进详情。
3. **模块2 · 推荐开发者**：更轻的卡片，突出「主阵地 + 为什么推荐」，每张带「纳入关注」动作（MVP 阶段为标记，落库手动）。
4. **模块3 · 今日热门开源项目**：横向滚动/列表，每条 = 名称 + 作者 + 一句介绍 + star 数 + 趋势 sparkline + 来源标签。作者若已在模块1，显示关联徽标。
5. **模块4 · 今日热门文章/教程**：列表，每条 = 标题 + 类型徽标 + 作者 + 一句「为什么有用」。

## 开发者详情页结构（模块1 核心）

- 头部：头像 / 名字 / oneliner / tier / 全部阵地链接（github/x/website/substack/youtube）
- 简介 summary
- 事迹 story
- 代表作 highlights（要点列表）
- 开源 & 商业项目（从 project_ids 关联渲染，带 star）
- 商业化分析 commercialization（模式 + 信号）
- 动态时间线 dynamics（cron append，倒序）
- 我的判断 my_take + 复刻意向 replicate_intent 徽标

## 筛选与搜索（渐进增强，非 MVP 必须）

- 标签 tag 筛选（跨模块通用）
- tier / status 筛选
- 全文搜索（前端 fuse.js 级别即可）
- 模块3 排序：star 数 / 新增 star / 发现时间

## MVP 边界

MVP = 首页 4 模块 + 三类详情页，读静态 JSON，无搜索无后台。先把 Zara 的真实数据渲染出来跑通，再叠加筛选和趋势图。
